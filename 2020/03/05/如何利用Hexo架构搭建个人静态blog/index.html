如何利用Hexo架构搭建个人静态blog

最近花时间搭建了个人静态blog，打算用来记录自己的学习过程，总结心得。现在打算先把搭建过程简单的记录一下。

Hexo 简介

Hexo是基于Node.js技术的静态博客架构，可以生成静态页面部署在不同的平台上，是搭建静态blog的不二之选。

感兴趣的朋友可以去Hexo官网进行了解、使用、研究。

搭建步骤

安装Git

简介：

Git是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。由 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件，它采用了分布式版本库的方式，不必服务器端软件支持。

安装Git，以后可以用git bash 用来管理Hexo文章，并且上传到代码托管平台。

具体功能推荐大家两个网站来了解:

廖雪峰老师的Git教程

菜鸟教程Git教程

下载链接：

Git官网

Tips:可以使用 git --version 查看是否安装成功和相应的版本号。

安装Node.js

Hexo基于Node.js技术，需要安装Node.js(里面包含npm工具)。

下载地址：Node.js官网 建议LTS版本进行下载使用的。

Tips：可以使用以下命令检查是否安装成功。

安装Hexo

首先，需要建立一个blog空白文件夹，在这个文件夹下，鼠标右键git bash here 打开终端。

Tips:npm访问资源太慢的话，可以安装cnpm提高下载资源速度，在终端输入

npm install -g cnpm --registry=https://registry.npm.taobao.org

以后下载用到npm的地方可以尝试使用cnpm替换下载。

安装Hexo

命令行：cnpm install -g hexo-cli

Tips：使用命令hexo -v测试是否安装成功。

建立项目

命令：hexo init

成功后，所在文件夹文件目录如下：

  • node_modules: 依赖包
  • public：存放生成的页面
  • scaffolds：生成文章的一些模板
  • source：用来存放你的文章
  • themes：主题
  • _config.yml: 博客的配置文件

托管代码平台创建个人仓库

一般代码托管平台大家会选择：

  • Github
  • coding
  • gitee

这单个平台都有相关的page服务，大家可以根据自己喜好来选择。应为网络访问的原因，我最终选择使用了gitee,来搭建个人blog。

在相关平台上创建以自己名字命名的仓库，后面加类似于xxx.gitee.io,其中xxx就是自己的名字。

ssh，就是一个秘钥。

  • id_rsa是你这台电脑的私人秘钥，不能给别人看的
  • id_rsa.pub是公共秘钥，可以随便给别人看。
